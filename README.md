# SEA FAMILLE

沖縄のマリンレジャーサービスを想定した、架空のコーポレートサイトです。  
HTML / CSS / JavaScript で静的実装し、VPS への公開、HTTPS 化、画像最適化まで行いました。

本作品は学習およびポートフォリオ用途で制作した架空サイトです。

## 概要

このサイトは、沖縄の海をテーマにしたマリンレジャー事業者を想定して制作したポートフォリオ作品です。  
はじめて訪れた人にも雰囲気やサービス内容が伝わるように、ヒーロー、サービス紹介、ギャラリー、お知らせ、CTA までを一貫したトーンで設計・実装しました。

## 工夫した点

本制作では、単に見た目を再現するだけでなく、実務での運用を想定した設計を意識しました。

まず、HTML / CSS / JavaScript の役割を明確に分離し、スタイルの管理を

- common（共通）
- layout（全体構造）
- components（部品）
- page（ページ固有）

に分割することで、保守性と再利用性を高めました。

また、UI面では初見ユーザーでも直感的に操作できるように、

- ハンバーガーメニュー
- ギャラリーモーダル
- スクロール演出（泡アニメーション）

などを実装し、視覚的な体験価値の向上を意識しました。

さらに、パフォーマンス面では画像の容量最適化に取り組み、

- 画像サイズの適正化（最大幅の調整）
- JPG / WebP への変換
- 不要な容量の削減

を行うことで、読み込み速度の改善を図りました。

加えて、制作したサイトを実際にVPSへデプロイし、

- Nginxによる公開設定
- 独自ドメインの紐付け
- HTTPS化（Let's Encrypt）

まで対応することで、実務と同等の公開フローを一通り経験しました。

これらを通じて、単なるコーディングにとどまらず、「設計・実装・公開」までを一貫して行う力を意識して制作しました。


## 難しかった点
本制作では、学習段階として
HTML5 の基礎（Webクリエイター能力認定試験エキスパート相当）までの知識をベースに、実務を想定した制作フローへ移行した点が大きな課題でした。

実務に近い形で進めるため、AIを補助ツールとして作成した簡易的なカンプ（Figma相当）をもとにHTML実装を行いましたが、設計の正解が明確でない中で「この構造が適切か」を都度検証しながら進める必要がありました。

特に、1セクションごとに

- 適切なHTML構造（セマンティックなタグ選定）
- CSSの責務分離
- JavaScriptの役割範囲

を意識しながら実装する過程で、理解と実装を同時に進める必要があり、表現として定着させるまでに時間を要しました。

また、HTML / CSS / JavaScript の責務分離においては、スタイルや挙動の競合が発生しやすく、意図しない動作の原因特定と修正に時間がかかりました。

これらの課題に対しては、

- クラス設計の見直し
- スタイルの責務整理（common / layout / page）
- JavaScriptの役割の明確化

を行うことで、最終的には保守性を意識した構成へ改善しました。


## 今後の改善案

今後は、ユーザー体験と運用面の両方を意識しながら、以下の改善に取り組みたいと考えています。

まず、お問い合わせ導線については、現在はCTA中心の設計となっているため、ページ内の各セクションに適切な導線を設けることで、ユーザーの行動をより自然に促せる構成へ改善したいと考えています。

また、現在はLPを意識したシングルページ構成のため、今後は下層ページ（プラン詳細・会社概要・よくある質問など）を追加し、サイト全体としての情報設計を強化していきたいです。

SEOの観点では、

- メタタグの最適化
- OGP設定によるSNSシェア時の見え方の改善
- 構造化データの導入

などを行い、検索・外部流入を意識した設計へ発展させたいと考えています。

さらにパフォーマンス面では、Lighthouseを指標として

- 画像の遅延読み込み（lazy loading）
- 不要なCSS / JavaScriptの削減
- 読み込み順序の最適化

などを行い、表示速度とユーザー体験の向上を図りたいです。

加えて、現在のデザインをベースにしながら、複数ページへ展開した際にも統一感を維持できるよう、デザインルールの整理とコンポーネント設計の強化にも取り組んでいきたいと考えています。
